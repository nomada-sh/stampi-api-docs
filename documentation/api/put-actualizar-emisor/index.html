[PUT] - Actualizar Emisor

Para actualizar un emisor (issuer), debe proporcionar en los encabezados API-ID, API-Key y CompanyID perteneciente, al igual que los archivos .cer y .ker con su contraseña correspondiente

Header Parameters
API-ID string — REQUIRED

Generada por Stampi

API-Key string — REQUIRED

Generada por Stampi

CompanyID uuid — REQUIRED

Hace referencia al identificador de la compañía a la que pertenece el usuario.

Request Body
certFile base64

Archivo .cer en base 64

keyFile base64

Archivo .key en base 64

password string

Contraseña del archivo .key

tradeName string

Nombre comercial

businessName string

Razón social

taxId string

RFC (Registro Federal de Contribuyentes)

taxRegimeId uuid

Se refiere al tipo de régimen en el cual esta registrado el cliente. Para este campo es necesario que vea nuestro catálogo de régimen fiscales para extraer el id que este campo requiere

address string

Se refiere al dirección que tiene registrada en el sat el cliente (fisica o moral)

postalCode string

Código postal

lastFolio string

Possible values: [0]

Se refiere al último folio de factura timbrada, en caso de que no tenga facturas asociadas a este RFC dejar en 0

lastFolioPago string

Possible values: [0]

Pago del último folio utilizado. En caso de que no tenga facturas asociadas a este RFC dejar en 0

cfdiVersion string

Possible values: [3.3, 4.0]

Se refiere a la versión del CFDI (3.3, 4.0)

pagoVersion string

Possible values: [1.0, 2.0]

Se refiere a al complemento de pago (Version 1.0, Version 2.0)

cancelacionVersion string

Possible values: [3.3, 4.0]

Se refiere a la versión del CFDI (3.3, 4.0) cancelación

Responses
201

Success Response
